常用 Hexo 指令

Posted on 2026-04-20

📝 写文章

新建文章：hexo new “文章标题”

或者在 source/_posts/ 目录下创建 .md 文件（或删除示例文章 hello-world.md），
然后重新运行 hexo clean; hexo generate; hexo deploy 即可更新博客。

如果想保留草稿不发布，可以把文章放在 source/_drafts/ 目录，或者在前置声明中添加 published: false。

🎨 更换主题

更换主题：Hexo 有大量精美主题（如 NexT、Butterfly、Stellar 等），安装后修改 _config.yml 中的 theme 字段即可。更新：cd themes/next，git pull;注意：更换主题后，需要重新生成并部署：hexo clean; hexo generate; hexo deploy

🌐 绑定自定义域名

购买域名（如从阿里云、腾讯云、Namecheap 购买）。

添加 CNAME 文件：在博客的 source 目录下手动创建一个名为 CNAME 的文件（无后缀），内容只写你的域名，例如：www.yourname.com
GitHub Pages 设置：
进入仓库 Settings → Pages
在 Custom domain 处填写你的域名，保存。
勾选 Enforce HTTPS（GitHub 会自动申请证书）。
域名解析：在你的域名控制台添加两条 CNAME 记录：
记录类型 主机记录 记录值
CNAME @ 你的用户名.github.io
CNAME www 你的用户名.github.io

💬 配置评论系统

推荐两款适合静态博客的评论系统：

Giscus（基于 GitHub Discussions）
优点：无广告，支持 Markdown，与 GitHub 账号打通，无需自建数据库。
配置步骤：
确保你的博客仓库开启了 Discussions（Settings → General → Features → 勾选 Discussions）。
安装 Giscus App 到你的仓库。
访问 giscus.app，填写仓库信息，生成配置代码。

将生成的 script 代码粘贴到主题的评论配置中（NexT 主题在 themes/next/_config.yml 中搜索 giscus 并开启）。

Valine（基于 LeanCloud）
优点：国内访问快，无需 GitHub 账号。
配置步骤：
注册 LeanCloud 国际版，创建应用。
获取 appId 和 appKey。
在主题配置文件中开启 Valine 并填入对应的 Key。
